# Active Learning Model Evaluation Framework

This directory contains the comprehensive evaluation framework used to assess the performance of object detection models (YOLO, RT-DETR, Faster R-CNN) dynamically trained during the Active Learning phases.

The evaluation process measures both standard bounding box detection metrics and holistic image-level binary filtering metrics across different iterations (Cycles 0 through 4) and subsets (Val and Test).

## Workflow: Two-Tiered Evaluation

1. **Image-Level (Binary Classification)**: Evaluates the model's fundamental ability to flag whether an animal is present, serving as a filter for unlabelled data.
   - **Methodology**: Aggregate the highest confidence bounding box per image for the target class (WLT or Class-Agnostic).
   - **Metrics**: **ROC-AUC** and **Optimal F1-Score** (calculated via confidence threshold sweeping), accompanied by Precision, Recall, and Confusion Matrices at the optimal threshold.
2. **Detection-Level (Instance Localization)**: Evaluates the specific bounding box accuracy strictly based on COCO standards.
   - **Metrics**: **mAP50** and **mAP50-95** globally, as well as Class-specific Average Precisions (AP50) for Western Leopard Toads, Small Mammals, and Other Amphibians. Computed using `torchmetrics`.

## Unified Evaluation Pipeline

The evaluation pipeline decoupled heavy inference from metrics calculation and plotting. This allows easy reprocessing of metrics without having to wait hours for batched GPU inferences.

### Modular Components

- **[`eval_utils/config.py`](eval_utils/config.py)**: Centralized configuration containing hyper-parameters, hardware variables (batch size, device), and dataset paths.
- **[`pipelines/run_inference_all.py`](pipelines/run_inference_all.py)**: Core execution script to run bulk inferences. Iterates over all active learning Phase 2 checkpoints (Cycles 0-4) and dynamically switches logic between PyTorch models (Faster R-CNN) and Ultralytics (YOLO/RT-DETR). Includes a parallel MegaDetector evaluation baseline. Inherits the exact `ActiveLearningInferenceDataset` (with CLAHE formatting, scaling) used during training.
- **[`pipelines/apply_spatial_filter.py`](pipelines/apply_spatial_filter.py)**: Processes all generated `_raw.json` inference outputs and applies spatial logic to clean up static false positive predictions.
- **[`pipelines/evaluate_metrics.py`](pipelines/evaluate_metrics.py)**: Core evaluation engine. Iterates through the filtered JSON files, matches them against the Ground Truth files from the Test/Val splits, and calculates all Image-level sweeps and Detection-level metrics, caching them to CSVs.
- **[`reporting/plot_evaluation_results.py`](reporting/plot_evaluation_results.py)**: Visualization generator that creates PR curves, bounded ROC-AUC plots, and confusion matrices for the target subset, along with aggregating everything into a final Markdown report.

## Usage

You can run the entire pipeline manually step-by-step to track the progress across your environments.

**1. Run Bulk Inference:**
This loads each model into the GPU, processes all validation and test images dynamically, and outputs `_raw.json` files for every combination.
```bash
python3 pipelines/run_inference_all.py
```

**2. Filter Predictions:**
This scans through all the generated predictions and applies the post-processing filter to prune static bounding boxes.
```bash
python3 pipelines/apply_spatial_filter.py
```

**3. Compute Metrics:**
Matches predictions with ground-truths and evaluates the COCO metrics alongside Image-Level ROC sweeps. Output CSVs and JSONs are placed in `results/files/`.
```bash
python3 pipelines/evaluate_metrics.py
```

**4. Generate Reports and Visualizations:**
Outputs PDF plots of ROC curves, PR Curves, and Confusion Matrices to `results/plots/`, and a comprehensive markdown summary report to `results/files/final_evaluation_results.md`.
```bash
python3 reporting/plot_evaluation_results.py
```

## Directory Organization

- `pipelines/`: Executable scripts responsible for iterating through the models, performing inference, filtering, and metric math.
- `reporting/`: Scripts used to consume the metrics and plot graphs.
- `eval_utils/`: Shared utilities and centralized configurations.
- `results/`: Autogenerated.
  - `results/files/`: Aggregated metric CSVs, JSON data for plotting, and the final Markdown report.
  - `results/plots/`: PDF visualizations.
  - `results/<model_name>/`: Specific prediction dumps generated during inference.
- `weights/`: Holds the foundational model weights for MegaDetector (`md_v5a.0.0.pt`).
